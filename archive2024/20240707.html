消除Windows Backup引發的「STATUS_WAIT_1」問題

使用Windows傳統的備份與還原（Windows 7）進行排程、週期形式的備份，經過了很長的時間後（中間沒有任何的問題或錯誤），會突然開始不能處理備份的排程工作，無論你如何設定都會跳出一個名為：STATUS_WAIT_1的錯誤，備份資料的行為因此被中斷。

如何解決Windows Backup引發的0x80780166或0x80070001問題

這個STATUS_WAIT_1的錯誤如果仔細去翻閱它的錯誤代碼，將會以0x80780166或0x80070001呈現，錯誤訊息可能如下：

備份磁碟區 C: 失敗。

Windows Backup 將資料寫入備份目標時發生錯誤。

Windows Backup encountered an error when writing data to the backup target. (0x80780166)

Additional Information:
STATUS_WAIT_1 (0x80070001)

解決的方法很簡單，首先到Windows Backup把當前所有的歷史資料都刪除（DELETE），然後跑到你的備份目的磁碟下，應該可以發現一個名為WindowsImageBackup 的資料夾，請使用最高權限把整個資料夾強制刪除（DELETE），接著再去Windows Backup重新開啟備份程序即可。

這個問題在網路上有很多人均遭遇到，估計是微軟備份程式在處理VHD格式方面出問題，但總之透過這樣的處理後，我確定就可以回歸到正常的備份行為了。
