關於UiPath Orchestrator的無人值守設定

最近觸碰到UiPath Orchestrator的需求，進行了局部研究後將雲端設定放置在此，供給有需要的人查看。

設定Machine的執行期授權

此步驟為設定Machine的執行期授權，最重要的就是Production Unattended工作數目。

Step 1. 當你的機器連接上雲端UiPath Orchestrator，請到Tenant的Machines頁籤，找到你自己的Workspace機器後，點選Edit Machine。

Step 2. 接著請在Production (Unattended)的欄位輸入1然後按下Update，如下圖所示：

這個步驟如果沒有進行，你可能在設定Trigger的設定過程中，遭遇到這樣的錯誤訊息：

There are no Unattended runtimes configured on this machine.

啟用帳號的無人值守與登入權限

Step 1. 當你的帳號分連接上雲端UiPath Orchestrator，請到Tenant的Manage Access頁籤，找到你自己的桌機使用者帳號後，點選Edit。

Step 2. 點選Personal automations setup頁籤，將Enable user to run automations勾選起來。

Step 3. 點選Unattended setup頁籤，將Enable this user to run unattended automations勾選起來。

Step 4. 接著在同一頁面的Credential usage on foreground automations選項，將Enable this user to run unattended automations點選，並在下列輸入你的Windows作業系統的登入帳號密碼，最後按下Update，如下圖所示：

這個步驟如果沒有進行，你可能在設定Trigger的設定過程中，遭遇到這樣的錯誤訊息：

Couldn't find any user with unattended robot permissions in the current folder. (#1671)
