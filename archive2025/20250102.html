GIT筆記：使用cli在10分鐘內快速學完遠端git指令（GitHub）

在本地端透過Git管理檔案，充其量只能管理檔案的版本，對於檔案的備份與交換其實毫無助益，因此我們通常會將本地端的檔案推送（push）到遠端的git server，或者是從遠端的git server拉下（pull）檔案到本地端。在本次的例子中，我們將採用GitHub來做為我們的git server。

建立GitHub的Repository

Step 1. 點選＋符號＞New repository。

Step 2. 輸入一些關於這個repository的相關資訊，可以選擇private私倉來隱藏一些資訊，捲到最下方按下Create按鈕就可以建立完成。

Step 3. 建立後系統會引導你到這個Repository的相關資訊頁面，其中位於畫面中間會有一個很像網址的資訊，分別以HTTPS、SSH標記，這個資訊很重要，等一下會用到。

建立本地端的實驗目錄

這一個步驟沒啥營養可說，基本上就是開立一個目錄，加入git管控，然後建立一個README.md的檔案，然後裡面撰寫一些markdown的資料後，將其git commit。

設定遠端Git資訊

Step 1. 設定origin的位址，請copy剛才GitHub的SSH位址後，將其輸入到下方指令：

git remote add origin {SSH位址}

Step 2. 設定透過HTTPS認證，請copy剛才GitHub的HTTPS網址後，將其輸入到下方指令：

git remote set-url origin {HTTPS網址}

推送本地端git到遠端GitHub

Step 1. 輸入下列指令將本地端檔案推送push到遠端GitHub。

git push -u origin master

Step 2. 由於第一次推送需要認證帳號密碼，因此git會彈出GitHub的認證視窗，請點選Sign in with your browser。

Step 3. 點選Authorize git-ecosystem。

Step 4. 接著回到剛才console畫面，就可以看到檔案被推送出去了。

Step 5. 回到遠端GitHub，可以看到我們編寫的README.md的內容已經在上面呈現了。

將遠端GitHub的資料拉回本地端

在多人共編的環境下，總會遇到遠端的檔案比本地端更新、或不一樣的問題，這時候我們會需要將遠端的檔案拉回pull來本地端進行更新或合併。

Step 1. 透過GitHub提供的線上編修檔案功能，將README.md取出並編輯內容，這樣一來將會形成遠端檔案比本地端新的狀況。

Step 2. 如果這時候本地端下了git push指令將會遭遇到錯誤警告，這時候我們就要採用git pull將檔案拉回來更新與合併，拉回後我們可以看到檔案的確已經更新成剛才線上編輯的內容了。

git pull origin master
