將.NET Core Console專案打包成單一執行檔

.NET Core已經推行數年了，但是他在一般使用者的電腦上普及率仍然很低，畢竟不會有一個普通使用者自己去下載.NET Core Runtime來安裝，反觀.NET Framework就沒有這樣的問題。因此，將.NET Core Console專案打包成單一執行檔就變成剛需，因為這樣就可以方便地在不同的環境中運行，而不需要安裝.NET Core SDK或Runtime。以下是如何使用dotnet publish命令來實現這個目標的步驟：

將.NET Core Console專案或DLL打包成單一執行檔

過程其實很簡單，到專案目錄(*.csproj)下打開命令視窗，執行以下命令即可：

dotnet publish -c Release -r win-x64 --self-contained true /p:PublishSingleFile=true /p:IncludeNativeLibrariesForSelfExtract=true

一般來說，發行通常會選用Release模式，如果你想要以Debug模式發行就請自己修改。這個命令會將專案編譯成一個單一的執行檔，並且包含所有必要的依賴項。這樣，你就可以在沒有安裝.NET Core Runtime的電腦上運行這個執行檔了。

以上面的範例，程式會自動在下列路徑下產生一個可執行檔，把這個exe檔案複製出來就可以獨立執行了：

\bin\Release\netX.X\win-x64
