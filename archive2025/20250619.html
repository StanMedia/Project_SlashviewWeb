規避Ubuntu的密碼變更原則

這幾天在進行Ubuntu Desktop的自動化腳本開發遇到了一個Ubuntu的密碼變更原則問題，在變更密碼的過程中系統會跟我報錯下列的錯誤訊息：

BAD PASSWORD: The password contains the user name in some form.
BAD PASSWORD: The password is too similar to the old one.
...

這種類似的問題好像在網路上還蠻多人發問的，甚至有人教你去覆寫shadow檔來解決這個問題，但我覺得這樣做不太好，因為這樣會破壞系統的安全性。

其實問題的解法很簡單，答案就是切到最高管理者root去變更密碼，然後不要鳥這些錯誤訊息持續輸入新密碼就變更成功了：

sudo passwd username

這個問題我在Ubuntu Server Minimal上面沒有遇到，但在Ubuntu Desktop上就遇到了，真的是蠻神奇的。
